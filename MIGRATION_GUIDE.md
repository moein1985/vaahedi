# راهنمای مهاجرت به Next.js

این سند مجموعه مراحل کوتاه را برای بازنویسی کامل پروژهٔ فعلی Node.js (Fastify + Vite/React) در پوشهٔ `vahedi-nextJS` با استفاده از Next.js شرح می‌دهد. هدف این است که تغییرات به صورت فازبندی شده و قابل مدیریت انجام شود و پروژهٔ اصلی بدون تغییر باقی بماند.

## فاز ۱ – ایجاد اسکلت Next.js

1. ساخت پوشهٔ جدید `vahedi-nextJS` در ریشهٔ workspace.
2. اجرای `npm init -y` و نصب `next react react-dom typescript` و ابزارهای پایه (eslint).
3. ایجاد فایل‌های پیکربندی: `next.config.js`, `tsconfig.json`, `package.json` (مثل نمونه موجود در راهنما).
4. افزودن ساختار پایه: پوشهٔ `pages/`، `styles/` و فایل `pages/_app.tsx`.
5. اجرای `npm run dev` برای اطمینان از اینکه اپ جدید بدون خطا بالا می‌آید.

> نتیجه: یک پروژهٔ Next.js خالی و مستقل داریم که آمادهٔ توسعه است.

## فاز ۲ – انتقال صفحهٔ اصلی و هدر

1. کپی‌پیست کامپوننت هدر و صفحهٔ اصلی (`index.tsx`) از `apps/web/src/routes` به `vahedi-nextJS/pages/index.tsx`.
2. اصلاح لینک‌ها به استفاده از `next/link` و حذف وابستگی‌های مربوط به `@tanstack/react-router`.
3. اضافه کردن استایل‌های پایه (از `styles/globals.css`) یا نگاشت Tailwind در صورت نیاز.
4. اجرای اپ Next.js و مشاهدهٔ صفحهٔ اصلی مطابقت با نسخهٔ قبلی.

## فاز ۳ – اشتراک‌گذاری کد

1. بررسی کدهای مشترک (کامپوننت‌ها، یوتیل‌ها، استایل‌ها) در `apps/web/src` یا `packages/shared`.
2. در صورت نبود چنین بسته‌ای، ایجاد `packages/ui` یا `packages/shared` با کامپوننت‌های قابل چندمصرف. `npm install ../../packages/…` از هر دو اپ استفاده شود.
3. انتقال تدریجی کامپوننت‌ها و یوتیل‌ها به این بستهٔ مشترک.
4. به‌روزرسانی importها در هر دو پروژه (Vite و Next) به مسیر بستهٔ مشترک.

## فاز ۴ – صفحات و مسیرهای اضافی

1. برای هر روت موجود (ورود، ثبت‌نام، داشبورد، پروفایل، کاتالوگ و غیره) یک صفحهٔ Next.js در `pages/` ایجاد کنید.
2. از داده‌های API فعلی (`apps/server`) با استفاده از `fetch` یا tRPC/next.js استفاده کنید. اگر tRPC مصرف می‌شود، نصب `@trpc/next` و تنظیم آن.
3. تست دستی هر صفحه را پس از توسعه انجام دهید و مطمئن شوید عملکرد مشابه نسخهٔ قبلی باشد.

## فاز ۵ – مهاجرت تست‌ها و CI

1. افزودن پایپ‌لاین‌های تست برای پروژهٔ Next.js (Vitest یا Jest) جداگانه.
2. در `turbo.json` یک تسک جدید `next:build` و `next:dev` تعریف کنید و آن را به جریان CI اضافه نمایید.
3. آزمایش E2E موجود (Playwright) را با آدرس `http://localhost:3001` یا متغیر `PLAYWRIGHT_BASE_URL` اجرا کنید.

## فاز ۶ – مهاجرت کامل و بازبینی

1. زمانی که همه صفحات، کامپوننت‌ها و تست‌ها به Next.js منتقل شدند، بازبینی کنید چه چیزهایی هنوز در `apps/web` لازم است.
2. بسته به تصمیم تیم، می‌توانید اپ قدیمی را نگه‌دارید یا حذف کنید.
3. اطمینان حاصل کنید که پوشهٔ جدید هیچ‌گاه وابستگی به کد Node.js ندارد (فقط API‌ها را مصرف کند).

> نکته: همیشه قبل از حذف یا تغییر ساختار بزرگ، یک برنچ جدید گیت بسازید و همهٔ تغییرات را به‌صورت incremental اعمال کنید.

با دنبال کردن این فازها، می‌توانید پروژه را بدون «سنگینی» به Next.js منتقل کنید و همزمان نسخهٔ قبلی را سالم نگه دارید.
