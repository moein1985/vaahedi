# شرح خدمات، دامنه تعهد و جمع بندی تجاری پروژه Vaahedi

## هدف سند
این سند برای ارائه به کارفرما تهیه شده و سه هدف را همزمان پوشش می دهد:
- تشریح روشن خدمات و اقلام قابل تحویل پروژه
- مشخص کردن مرز تعهدات قراردادی و موارد خارج از دامنه
- ارائه جمع بندی زمانی و مالی در سطح مدیریتی

## خلاصه مدیریتی

| موضوع | جمع بندی پیشنهادی |
|---|---|
| ماهیت پروژه | توسعه و تحویل یک پلتفرم B2B تجارت خارجی با پنل کاربر، پنل ادمین، مدیریت محصولات، درخواست های تجاری، گفت وگو و خدمات زیرساختی |
| مدل تحویل | توسعه از صفر تا نسخه عملیاتی نزدیک به وضعیت فعلی پروژه |
| برآورد زمانی کل | حدود 720 تا 1150 ساعت |
| بازه زمانی اجرایی | حدود 12 تا 16 هفته کاری برای یک تیم کوچک یا یک توسعه دهنده فول استک با پشتیبانی محدود |
| برآورد قیمت فروش محصول | نمونه محاسباتی: حدود 12,000,000,000 ریال |
| هزینه بهره برداری سال اول | مستقل از قیمت فروش محصول و وابسته به زیرساخت، SSL، نگهداری و سرویس های ثالث |
| وضعیت OTP پیامکی | به عنوان ماژول توسعه ای مستقل یا آپشن قراردادی پیشنهاد می شود |
| مبنای فنی | React + TypeScript + Vite + Node.js + Fastify + tRPC + Prisma + PostgreSQL + Redis + MinIO + BullMQ |

## اقلامی که حتما باید در قرارداد با صراحت ذکر شوند

| موضوع | متن پیشنهادی برای شفاف سازی |
|---|---|
| تفکیک قیمت محصول و عملیات | قیمت فروش یا توسعه محصول مستقل از هزینه های زیرساخت، دامنه، SSL، سرور، سرویس پیامکی و نگهداری است. |
| مدیریت تغییرات | هر نیازمندی جدید یا تغییر در دامنه، پس از بررسی Change Request و توافق زمان و هزینه انجام می شود. |
| سرویس های ثالث | اختلال، محدودیت، تغییر قیمت یا تغییر سیاست سرویس دهندگان ثالث خارج از کنترل پیمانکار است. |
| کیفیت و پذیرش | معیار پذیرش بر اساس سناریوهای توافق شده تست می شود، نه بر اساس انتظار ضمنی یا موردی خارج از دامنه. |
| پشتیبانی | پشتیبانی شامل توسعه قابلیت جدید نیست و فقط طبق SLA شامل رفع ایرادهای نسخه تحویلی است. |
| مالکیت و تحویل | نوع واگذاری باید صریح باشد: واگذاری کامل سورس، حق بهره برداری، یا واگذاری محدود همراه با خدمات پشتیبانی. |
| داده و محتوا | تامین محتوای نهایی، لوگو، متن صفحات، قوانین کسب وکار، اطلاعات ادمین و داده های پایه با کارفرما است مگر خلاف آن تصریح شود. |
| امنیت و بکاپ | بکاپ، بازیابی، سیاست نگهداری، سطح امنیت و مسئولیت های هر طرف باید جداگانه مشخص شود. |

## موارد خارج از دامنه مستقیم مگر با توافق جداگانه

- توسعه اپلیکیشن موبایل بومی
- یکپارچه سازی با ERP، CRM، حسابداری یا سرویس های اختصاصی سازمانی
- گزارش ساز تحلیلی بسیار پیشرفته یا داشبوردهای BI سفارشی
- سئو پیشرفته، تولید محتوا، تبلیغات و دیجیتال مارکتینگ
- مهاجرت کامل داده از سامانه های قدیمی
- تضمین کیفیت خروجی مدل های هوش مصنوعی یا تضمین دسترس پذیری دائمی سرویس های خارجی
- راهبری دائمی سرور، مانیتورینگ 24 ساعته و عملیات DevOps مستمر مگر در قالب قرارداد نگهداری

## پیش نیازهای لازم از سمت کارفرما

- تایید دامنه نهایی پروژه و زیر دامنه های مورد نیاز
- ارائه دسترسی سرور یا خرید زیرساخت مورد تایید
- تعیین نقش ها، سطوح دسترسی و فرایندهای کسب وکار اصلی
- تعیین محتوای نهایی صفحات عمومی، اخبار، اسناد و اعلان ها
- ارائه کلیدها و اعتبار سرویس های خارجی در صورت نیاز
- تایید سناریوهای پذیرش پیش از فاز تحویل

## جمع بندی نهایی برای ارسال

- این پروژه در سطح یک وب سایت MVP ساده نیست و باید به عنوان یک پلتفرم عملیاتی چندماژوله قیمت گذاری و قرارداددهی شود.
- پیشنهاد می شود قرارداد بر اساس دامنه تعهد، معیار پذیرش، اقلام خارج از قرارداد و خدمات آپشنال تنظیم شود.
- OTP پیامکی، سرویس های ثالث، دامنه، SSL و بخشی از عملیات زیرساخت بهتر است به صورت الحاقیه یا ردیف مستقل مالی دیده شوند.
- اگر هدف، فروش محصول آماده باشد، قیمت فروش محصول باید از هزینه سال اول بهره برداری جدا نوشته شود.
