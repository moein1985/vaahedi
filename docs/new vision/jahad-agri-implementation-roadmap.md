## واقعیت فنی محصول فعلی (As-Is)
- وب اپ فعلی با React، Vite، TanStack Router، TanStack Query، Tailwind CSS v4 و tRPC Client پیاده سازی شده است؛ بنابراین برنامه اجرای UI باید با همین stack پیش برود، نه Next.js App Router.

- در `packages/shared` و `packages/db` مدل ها و enumهای پایه مانند `UserRole`، `DocumentType`، `VerificationStatus` و `CommodityGroup` از قبل موجود هستند.
- جریان تایید مدارک و وضعیت بازبینی در پنل ادمین و shared schemaها وجود دارد و باید توسعه یابد، نه اینکه با سیستم موازی جایگزین شود.

- افزودن فیلترها بر اساس شغل، محصول، استان و نوع فعالیت
- انطباق فرم های ثبت محصول/خدمت با واژگان دامنه جدید
- انطباق ماژول `services` با خدمات تخصصی کشاورزی
- اجرای migration محتوایی برای labelها، i18n و SEO مرتبط با دامنه قدیمی

### خروجی ها
- taxonomy قابل مدیریت در UI و Backend
- فرم های ثبت محصول/خدمت دامنه کشاورزی
- جست وجو و فیلتر اولیه قابل استفاده

### Definition of Done
- کاربر بتواند آیتم خود را در دسته صحیح ثبت کند و با فیلترها آن را بازیابی کند.

## فاز 4: درخواست ها، فرصت ها و ارتباطات
هدف: عملیاتی کردن سناریوهای تعامل بین ذینفعان.

### کارها
- انطباق ماژول درخواست ها با سناریوهای خرید، فروش، واردات، صادرات و همکاری
- بازطراحی labelها و CTAها متناسب با تجارت کشاورزی
- بازبینی جریان پیام رسانی یا ثبت تعاملات در سناریوهای کلیدی
- افزودن فیلترهای کاربردی برای فرصت ها و درخواست ها

### خروجی ها
- ماژول درخواست/فرصت متناسب با دامنه جدید
- مسیرهای ارتباطی پایه بین ذینفعان

### Definition of Done
- سناریوی ثبت درخواست و مشاهده پیشنهادها در UI قابل انجام باشد.

## فاز 5: تقویم برداشت و تحلیل بازار نمایشی
هدف: ارائه لایه آگاه سازی قابل کنترل برای تصمیم یاری غیرتعهدی.

### کارها
- تعریف ساختار داده تقویم برداشت (محصول، منطقه، بازه)
- طراحی ویجت/صفحات نمایشی برای تقویم برداشت
- طراحی بخش تحلیل بازار نمایشی با داده کنترل شده
- درج disclaimer روشن درباره ماهیت اطلاع رسانی این بخش

### خروجی ها
- بخش تقویم برداشت قابل نمایش
- بخش تحلیل بازار نمایشی در داشبورد/صفحه اختصاصی

### Definition of Done
- کاربر داده نمایشی را ببیند و متن عدم تعهد به شکل واضح نمایش داده شود.

## فاز 6: پنل ادمین، محتوا و عملیات
هدف: توانمندسازی عملیات روزمره بدون مداخله فنی مستقیم.

### کارها
- مدیریت taxonomy مشاغل و محصولات در پنل ادمین
- مدیریت آیتم های تقویم برداشت و تحلیل بازار
- مدیریت صف بازبینی مجوزها
- گزارش های پایه: تعداد کاربران، مجوزهای در انتظار، درخواست های فعال

### خروجی ها
- پنل ادمین عملیاتی برای MVP
- ابزار مدیریت محتوا و اعتبارسنجی اولیه

### Definition of Done
- ادمین بتواند taxonomy، محتوا و وضعیت های بازبینی را مدیریت کند.

## فاز 7: AI کمکی و بهبود تجربه کاربری (اختیاری در MVP)
هدف: افزودن قابلیت کمکی با دامنه کنترل شده و ریسک پایین.

- بهبود داشبورد کاربر با مسیرهای سریع مبتنی بر نقش/فعالیت

### خروجی ها
- AI کمکی محدود و قابل کنترل
- UX یکدست تر برای شروع استفاده از سامانه

### Definition of Done
- قابلیت AI نقش کمکی داشته باشد و تصمیم قطعی تولید نکند.

## فاز 8: تست، سخت سازی و آماده سازی انتشار MVP

### خروجی ها
- MVP قابل دمو و قابل استقرار
- مسیرهای بحرانی دارای تست عبوری

### Definition of Done
- سناریوهای اصلی پاس شده باشند و blocker بحرانی باز نمانده باشد.
